前言

为了实现项目代码的规范管理，提高代码的可维护性，增强不同团队的协作能力，在进行项目推进会议中，一致统一并编写了本项目开发编码规范。

整体规范

编码规范整体参照:阿里巴巴java开发规范

业务分包模式

以模块进行分包，分包下面包含controller、mapper、service。

类注释

包含：创建人、日期、基本功能描述

方法注释

包含：人（非类创建人增加方法时）、日期、参数、描述、处理流程

swagger2[强制]

swagger2对新模块功能进行强制增加组件。

1.增加编码的规范性；

2.增加前后端的对接能力；

3.增加其他人其他模块对接对于平台整体接口的了解。

config配置【基于springcloud版本】

新增配置：

​ 各个目录（如：利津、东营），去增加相关参数；
​ 注释内容：功能、日期、联系人

对应的java代码

要增加相关的检测跟提示，非必须的设置一个默认值；

通过相关参数获取通过get方法进行获取，在get方法中增加为空检测，如果为空打印error级别提示，并抛出异常。

idea插件【建议】

建议安装idea插件：阿里巴巴规约插件

规避魔法值

非必须情况，减少魔法值，通过字典、配置、常量类实现。

svn/git提交规范

提交步骤【最重要】

  • 提交代码务必先更新拉取再提交，先更新拉取再提交！！重要的事情说五遍！
  • 更新的代码务必是可以运行无错误的代码。

新增
